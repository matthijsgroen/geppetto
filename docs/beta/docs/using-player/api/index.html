API Reference

The api has a only a few methods to get you started:

  • use setupWebGL or createPlayer to initialize the canvas and get an GeppettoPlayer object.
  • use prepareAnimation to convert your animation to use with GeppettoPlayer.addAnimation.
  • use the AnimationControl (as return of GeppettoPlayer.addAnimation) to control your animation.

setupWebGLâ€‹

Initializes the WebGL Context of a provided context. Configures the context and returns a GeppettoPlayer bound to this element.

Use this method if you only render Geppetto Animations in your Canvas. Use createPlayer if you want your own control over the canvas configuration

Returns GeppettoPlayer

Argumentsâ€‹

  • element HTMLCanvasElement the Canvas DOM element that is not yet initialized with a context
const setupWebGL = (element: HTMLCanvasElement): GeppettoPlayer;

createPlayerâ€‹

Initializes a player to display in an existing WebGL Environment. Use this function to create a player if you want to have full control over the rendering process (possibly to combine with other render code).

Returns GeppettoPlayer

Argumentsâ€‹

  • element HTMLCanvasElement the Canvas DOM element containing a WebGL Context
const createPlayer = (element: HTMLCanvasElement): GeppettoPlayer;

prepareAnimationâ€‹

Convert the JSON based input animation file into a preprocessed list of buffers to place into WebGL

Returns PreparedImageDefinition

Argumentsâ€‹

  • imageDefinition ImageDefinition 
const prepareAnimation = (imageDefinition: ImageDefinition): PreparedImageDefinition;

GeppettoPlayerâ€‹

A player to add Gepetto animations to.

destroy()â€‹

Destroys all animations added to this player.

Returns void

addAnimation(animation, image, textureUnit, [options])â€‹

Add a Geppetto animation to the player.

Returns AnimationControls

Argumentsâ€‹

  • animation PreparedImageDefinition an animation prepared with prepareAnimation.
  • image HTMLImageElement a HTML Image element with loaded url to use as texture.
  • textureUnit number The texture unit to use you can use 0 for your first animation, 1 for your second, etc.
  • options Partial<AnimationOptions> 

/**
 * Options to set directly when adding an animation.
 **/
interface AnimationOptions {
  
  /**
   * Horizontal position of image in canvas. `0` = center, `-1` = left, `1` = right.
   * @default 0.0
   **/
  panX: number;
  
  /**
   * Vertical position of image in canvas. `0` = center, `-1` = bottom, `1` = top.
   * @default 0.0
   **/
  panY: number;
  
  /**
   * Adds a stacking order to the rendering elements, this helps when
   * stacking multiple animations on top of eachother.
   * @default 0
   **/
  zIndex: number;
  
  /**
   * Zoom level. `1` = 100%, `1.5` is 150%, `0.5` = 50% zoom.
   * @default 1.0
   **/
  zoom: number;
};

render()â€‹

Clears the canvas. Use this when you created the player with setupWebGL. If you want to control the rendering process (and the clearing of the canvas) yourself, skip the call to this method in your render cycle.

Returns void

Options to control the animation, start animation tracks, etc.

Clears all memory associated to this animation.

Returns void

getControlValue(controlName)â€‹

Retreives current value of a control. This value will not update for each frame of an animation. It will only update at the end of each play iteration of an animation.

Returns number

Argumentsâ€‹

  • controlName string name of the control to get value from

Render a frame of the image.

setControlValue(controlName, value)â€‹

Manipulates a control. Will stop animations that are using this control as well.

Returns void

Argumentsâ€‹

  • controlName string name of the control to change
  • value number value to set for control. Take into account that each control can have different value limits, depending on the amount of step a control has.

setLooping(loop, trackName)â€‹

Set the looping state of an animation track.

Returns void

Argumentsâ€‹

  • loop boolean true for looping, false to stop looping.
  • trackName string the name of the animation track to adjust.

setPanning(panX, panY)â€‹

Update the panning of the animation.

Returns void

Argumentsâ€‹

  • panX number value of horizontal panning. 0 = center, -1 = left, 1 = right.
  • panY number value of vertical panning. 0 = center, -1 = bottom, 1 = top.

setZIndex(zIndex)â€‹

Changes the rendering order of animations.

Returns void

Argumentsâ€‹

  • zIndex number The index number for rendering. The higher the number, the more in the front the element will be stacked

setZoom(zoom)â€‹

Updates the zoom level.

Returns void

Argumentsâ€‹

  • zoom number 1 = 100%, 1.5 is 150%, 0.5 = 50% zoom.

startTrack(trackName, [options])â€‹

Start an animation. Conflicting animations will be automatically stopped.

Returns void

Argumentsâ€‹

  • trackName string the name of the animation track to start. If the name is not valid, an exception will be thrown indicating what animation names are available.
  • options PlayOptions 
type PlayOptions = {

  /**
   * Start animation at given ms.
   **/
  startAt: number;
};
